# Codemania

Мобильная Android-платформа для IT-образования: практические задания, курсы, расписание, чат с менторами и отслеживание прогресса.

## Цель проекта

Сделать путь разработчика эффективным и практико-ориентированным:

- развитие навыков через практику
- единая экосистема для ученика и ментора
- автоматизация обучения и контроля прогресса

## Технологии

| Категория | Стек |
|-----------|------|
| Язык | Kotlin |
| Интерфейс | Jetpack Compose |
| Архитектура | Clean Architecture, MVVM, Repository |
| Локальные данные | Room |
| Сеть | Retrofit, REST API, JSON |
| Внедрение зависимостей | Hilt |
| Навигация | Navigation Compose |
| Дизайн | [Figma](https://www.figma.com/design/RMObkad4KsRfy4yVKTnuj1/Codemania) |

## Структура проекта

```
codemania/
├── app/                    # точка входа, DI, root-навигация
├── core/
│   ├── common/             # Result, расширения, диспетчеры
│   ├── network/            # Retrofit, перехватчики
│   ├── database/           # база Room
│   ├── ui/                 # общие UI-компоненты
│   └── designsystem/       # цвета, типографика, кнопки
├── domain/                 # бизнес-логика (без Android-зависимостей)
│   ├── model/              # доменные модели
│   ├── repository/         # интерфейсы репозиториев
│   └── usecase/            # сценарии использования
├── data/                   # слой данных
│   ├── remote/             # API, DTO
│   ├── local/              # DAO, сущности
│   ├── mapper/             # маппинг dto/entity → domain
│   └── repository/         # реализации репозиториев
└── feature/                # UI-модули по экранам
    ├── auth/               # вход / регистрация
    ├── catalog/            # каталог курсов
    ├── cart/               # корзина
    ├── course/             # карточка курса
    ├── lesson/             # задания, сдача решений
    ├── schedule/           # расписание
    ├── chat/               # чат с менторами
    └── profile/            # профиль, мои курсы
```

### Зависимости между модулями

```
feature:*  →  domain, core:ui, core:designsystem
data       →  domain, core:network, core:database, core:common
domain     →  core:common
app        →  все feature-модули + data
```

## Экраны

| Модуль | Что внутри |
|--------|------------|
| `auth` | Вход, регистрация |
| `catalog` | Список курсов, поиск, фильтры по языку |
| `cart` | Корзина, оформление |
| `course` | Описание курса, менторы, план занятий |
| `lesson` | Задание, комментарий, прикрепление файла, отправка |
| `schedule` | Расписание на день, просроченные задания |
| `chat` | Список менторов, переписка |
| `profile` | Профиль, мои курсы, рекомендуемые материалы |

## Доменные модели

| Модель | Описание |
|--------|----------|
| `User` | Пользователь (роль: ученик / ментор) |
| `Course` | Курс: название, теги, цена, описание |
| `Lesson` | Занятие: задание, дедлайн, статус |
| `Submission` | Сдача решения |
| `ScheduleItem` | Элемент расписания |
| `Chat`, `Message` | Чат и сообщения |
| `CartItem` | Позиция в корзине |
| `Material` | Рекомендуемый материал |

## API (черновик)

| Метод | Описание |
|-------|----------|
| `POST /auth/login` | Авторизация |
| `POST /auth/register` | Регистрация |
| `GET /courses` | Каталог курсов |
| `GET /courses/{id}` | Детали курса |
| `GET /courses/{id}/lessons` | Занятия курса |
| `POST /lessons/{id}/submit` | Отправка решения |
| `GET /schedule` | Расписание |
| `GET /chats` | Список чатов |
| `GET /chats/{id}/messages` | Сообщения |
| `POST /chats/{id}/messages` | Отправка сообщения |
| `GET /users/me` | Профиль |
| `GET /users/me/courses` | Мои курсы |

## План разработки

| Фаза | Период | Задачи |
|------|--------|--------|
| 1. Анализ и планирование | сен — нач. окт | Архитектура, REST API, вайрфреймы |
| 2. MVP-интерфейс | окт | Навигация, авторизация, Retrofit + репозитории |
| 3. Функциональность | ноя | Курсы и уроки, офлайн, роли, push-уведомления |
| 4. Релиз | дек | Тестирование, оптимизация, сборка |

## Запуск

> Проект в стадии инициализации. Конфигурация Gradle и код ещё не подключены.

```bash
# после настройки Gradle
./gradlew assembleDebug
```
