# Распределение задач

**Учебный проект** — мобильное приложение для защиты на курсе / в вузе.  
Цель не коммерческий релиз, а показать навыки команды: Kotlin, Clean Architecture, Compose, работа с API и командная разработка.

Команда: **Ромчик** + **Димон** (разработка) + **Фариза** (дизайн и контент).


|            | Ромчик                                                               | Димон                                         | Фариза                      |
| ---------- | -------------------------------------------------------------------- | --------------------------------------------- | --------------------------- |
| **Зона**   | Архитектура, ядро, обучение                                          | Сеть, данные, коммерция, чат, **организация** | Figma, контент, презентация |
| **Модули** | `app`, `core:`*, `domain`, `course`, `lesson`, `schedule`, `profile` | `data`, `auth`, `catalog`, `cart`, `chat`     | —                           |


**Принцип:** код — у Ромчика и Димона. Фариза — макеты, контент и презентация для защиты.

---



## Что сдаём на защиту


| Обязательно                   | По возможности              |
| ----------------------------- | --------------------------- |
| Рабочий debug APK             | Push-уведомления (FCM)      |
| 5–7 ключевых экранов по Figma | Загрузка файлов при сдаче   |
| Mock API (JSON / json-server) | CI на GitHub Actions        |
| Clean Architecture + MVVM     | ProGuard, release-сборка    |
| Room для кэша                 | Роли ученик / ментор        |
| Презентация + live demo       | Полная офлайн-синхронизация |


**Бэкенда нет** — данные из mock JSON или локального mock-сервера. Реальная оплата не нужна: корзина работает как UI-флоу.

### Минимальный demo-сценарий для защиты

1. Вход (можно с фиксированным логином)
2. Каталог курсов → карточка курса
3. Список занятий → экран задания
4. Расписание или профиль
5. Корзина или чат — на выбор команды

## Definition of Done

**Ромчик и Димон:**

- [ ] Экран собирается, данные приходят из mock
- [ ] PR смотрел второй разработчик
- [ ] Не падает на demo-сценарии

**Димон (организация):**

- [ ] Demo-сценарий проходится без сбоев
- [ ] Блокирующие Issues закрыты

**Фариза:**

- [ ] Макеты готовы до начала вёрстки экрана
- [ ] Контент передан Димону для mock JSON
- [ ] Презентация готова, репетиция проведена

**На защите показываем:**

- [ ] Архитектуру (слои, модули, зачем так) — отвечает Ромчик
- [ ] Стек (Kotlin, Compose, Room, Retrofit) — отвечает Димон
- [ ] Live demo по сценарию — Фариза ведёт, Димон держит телефон
- [ ] Figma-макеты — Фариза
